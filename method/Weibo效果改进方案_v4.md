# Weibo 数据集效果改进方案 v4

## 问题诊断

### 核心矛盾
在 weibo 数据集上，MCTL 系统面临的根本挑战是：**微博是社交媒体平台，真实和虚假内容在"信息完整性"和"写作风格"维���上差异很小**。

具体表现：
1. **工具层面**：knowledge_grounding 和 evidence_comparator 在 weibo 上的 FP 率（将 real 判为 fake）分别为 15.82% 和 11.64%，核心原因是它们用"正规新闻标准"评判社交媒体内容——"缺乏来源""描述笼统"在微���上是 real 和 fake 的共同特征
2. **Prompt 补丁的局限**：纯靠 prompt 补丁调节是一个精度-召回的跷跷板——放松判假标准提升了 real recall（71%→84%），但 fake recall 显著下降（94%→76%），MacroF1 反而降了
3. **检索先验噪声**：49% 的 real→fake 错误中检索到的相似样本多数为 fake，说明 CAMR 检索在 weibo 上存在系统性标签偏差

## 改进方案

### 策略：不再依赖单一 prompt 补丁，而��在架构层面做调整

### 改进1：检索标签感知的双阶段决策（核心改进）

**原理**：当工具判断和检索先验冲突时，不再简单二选一，而是根据冲突模式做差异化处理。

具体实现：在 CBDF 阶段增加以下逻辑（代码层面，非 prompt）：
- 如果**工具多数投 fake + 检索先验也偏 fake** → 直接判 fake（高置信）
- 如果**工具多数投 real + 检索先验也偏 real** → 直接判 real（高置信）
- 如果**工具多数投 fake + 检索先验偏 real** → 送入 CBDF LLM 仲裁（原流程）
- 如果**工具多数投 real + 检索先验偏 fake** → 送入 CBDF LLM 仲裁（原流程）

关键变���：对于一致的情况，不再调用 LLM（避免 LLM 被 prompt 补丁影响而翻转正确答案）；只在冲突时才让 LLM 仲裁。

### 改进2：回退 prompt 补丁为��量版

将 prompt 补丁从"强指导"改为"轻提醒"，避免过度干预 LLM 判断：
- 工具补丁：只保留"微博是社交媒体，信息不完整是正常的"这一句核心提醒
- 删除 MPRE 和 CBDF 的补丁（改由代码逻辑处理）

### 改进3：工具级别的��态置���度校准

对于中文社交媒体数据集，在工具输出后做置信度校准：
- 如果 `rhetorical_scanner` 或 `emotional_manipulator` 判定 fake，置信度乘以 0.5（在微博上这两个工具不可靠）
- 如果 `knowledge_grounding` 或 `evidence_comparator` 判定 fake 且置信度 > 0.8，保持不变（事实性工具的高置信判断可信）

### 改进4：CBDF 投票机制硬编码优化

在 CBDF 之前增加一个硬规则层：
- 如果 weighted_vote margin >= 0.8 且 有事实性工具参与 → 直接采用投票结果，不调 CBDF LLM
- 这避免了 LLM 在 CBDF 阶段"想太多"而翻转正确答案的问题（v2/v3 的 CBDF 翻转了工具全投 fake 的正确结果）

## 预期效果
- Real recall：保持 80%+ （通过轻量 prompt 提醒）
- Fake recall：恢复到 85%+（通过代码层面保护工具共识不被翻转）
- MacroF1：目标 0.83+
