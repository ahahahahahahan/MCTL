# MCTL 渐进式消融实验方案

## 核心思路

从最简单的 Baseline 出发，**每一步只加入一个模块**，通过对比前后效果来验证该模块的贡献。共设计 **6 个实验阶段**（1 个 Baseline + 5 个模块递增）。

---

## 实验阶段设计

### Exp0: Baseline — Direct LLM Prompting
> 不使用任何模块，纯 LLM 零样本判断

- **输入**: 原始新闻文本 + 图像（CLIP描述）
- **方法**: 直接给 LLM 一个简单 prompt："判断以下新闻是否为虚假新闻"
- **目的**: 建立性能下界，验证 LLM 裸推理的能力天花板
- **预期**: 准确率较低，因为缺乏外部知识和结构化推理

```
Pipeline: Input -> LLM Direct Prediction -> Output
```

---

### Exp1: + CAMR（上下文增强多模态检索）
> 在 Baseline 基础上加入检索模块

- **输入**: 原始新闻 + CAMR 检索到的 Top-K 相似样本上下文
- **方法**: 用 CLIP 嵌入检索相似新闻样本，将检索结果（文本、标签、事件上下文）拼接到 prompt 中，让 LLM 参考后判断
- **对比**: Exp1 vs Exp0 -> **验证检索增强的作用**
- **预期提升来源**: 解决"上下文稀疏性"问题，LLM 有了外部事实参照
- **关键超参**: K 值（检索数量），后续做参数敏感性分析

```
Pipeline: Input -> CAMR检索 -> LLM + Context Prediction -> Output
```

---

### Exp2: + MCP（元认知规划）
> 在 Exp1 基础上加入元认知规划模块

- **输入**: 原始新闻 + CAMR 上下文
- **方法**: LLM 先基于检索上下文进行 Chain-of-Thought 推理，**诊断该新闻需要哪些分析维度**（如：是否需要事实核查？是否存在图文冲突？是否有情绪煽动？），生成任务 DAG，然后按 DAG 规划的维度逐一分析后做判断
- **对比**: Exp2 vs Exp1 -> **验证元认知规划的作用**
- **预期提升来源**: 解决"分析异构性"问题，不同类型新闻自适应选择分析角度，而非千篇一律
- **注意**: 此阶段只用 MCP 规划 + LLM 自身逐维度分析，尚未引入专门工具

```
Pipeline: Input -> CAMR检索 -> MCP规划DAG -> LLM逐维度分析 -> Output
```

---

### Exp3: + ATR（自适应工具路由）
> 在 Exp2 基础上加入 8 个专门工具

- **输入**: 原始新闻 + CAMR 上下文 + MCP 规划的 DAG
- **方法**: 将 Exp2 中 LLM 的"自己逐维度分析"替换为**调用专门的 LLM-based 工具**：
  1. 语义解构器 (Semantic Dissector)：分析文本字面/隐含义
  2. 修辞探测器 (Rhetorical Scanner)：识别反讽、夸张、煽动性语言
  3. 知识锚定器 (Knowledge Grounding)：验证事实与检索事件一致性
  4. 期望偏移器 (Expectation Deviator)：计算图文预期偏差
  5. 跨模态对齐器 (Cross-Modal Aligner)：CLIP 嵌入相似度 + 语义一致性评分
  6. 不一致放大器 (Inconsistency Amplifier)：量化文本-图像-检索结果的矛盾点
  7. 情感操纵器 (Emotional Manipulator)：解析情绪操纵元素
  8. 证据对比器 (Evidence Comparator)：比对待检测项与检索样本的 label 偏移
- **ATR 调度**: 按 DAG 拓扑序执行工具，支持短路执行（confidence > 0.9 则跳过后续低优先级工具）
- **对比**: Exp3 vs Exp2 -> **验证专门化工具 vs 通用 LLM 分析的优势**
- **预期提升来源**: 专门化 prompt 比通用分析更精准；DAG 依赖保证逻辑完备性；短路机制提高效率

```
Pipeline: Input -> CAMR检索 -> MCP规划DAG -> ATR工具调度执行 -> 取最后工具输出作为判断 -> Output
```

---

### Exp4: + MPRE（多视角推理集成）
> 在 Exp3 基础上加入多视角推理集成

- **输入**: ATR 所有被执行工具的 (reasoning_trace, confidence_score)
- **方法**: 不再简单取最后一个工具的输出，而是让 LLM **综合所有工具的推理轨迹**：
  1. 识别共识证据（如多工具指出图文冲突）
  2. 仲裁冲突证据（如修辞工具说"夸张表述"，但知识工具说"事实错误"）
  3. 生成加权摘要，强调高置信度维度
- **对比**: Exp4 vs Exp3 -> **验证多视角融合的作用**
- **预期提升来源**: 多工具证据互相验证，减少单一视角片面性

```
Pipeline: Input -> CAMR -> MCP -> ATR工具执行 -> MPRE证据融合 -> Output
```

---

### Exp5: + CBDF（基于共识的决策融合）— 完整 MCTL
> 完整框架

- **输入**: MPRE 摘要 + CAMR 的检索标签分布
- **方法**: 在 MPRE 证据摘要基础上，引入检索样本的**标签分布**作为先验信号，进行不确定性感知决策：
  - 若共识摘要显式提及虚假新闻特征，且检索多数为虚假 -> 强虚假信号
  - 若工具置信度分歧 > 30%，则降级为"需人工审核"
- **对比**: Exp5 vs Exp4 -> **验证标签先验 + 不确定性校准的作用**
- **预期提升来源**: 检索标签分布提供统计先验；不确定性机制提高鲁棒性

```
Pipeline: Input -> CAMR -> MCP -> ATR -> MPRE -> CBDF -> Output（完整MCTL）
```

## 实验执行顺序建议

1. **先跑 Exp0 和 Exp1** — 验证检索是否有效，这是整个框架的基础
2. **再跑 Exp2** — 验证 MCP 规划是否比"直接分析"更好
3. **再跑 Exp3** — 这是工作量最大的一步（实现 8 个工具），确认工具化是否比通用分析强
4. **再跑 Exp4 和 Exp5** — 验证融合和决策模块
5. **最后** — 补充参数敏感性、LLM 底座鲁棒性、效率分析和 Case Study

> **关键原则**: 如果某一步加入模块后效果没有提升，应在进入下一步之前**先调优该模块**（调 prompt、调参数），确保每个模块都能带来正向贡献后再继续叠加。
