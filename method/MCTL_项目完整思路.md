# MCTL: Meta-Cognitive Tool-Learning Framework for Multimodal Fake News Detection

## 项目完整思路总览

> 本文档整合项目全部方案设计、代码实现、实验记录与改进策略，形成完整的项目技术脉络。

---

## 一、研究背景与动机

### 1.1 问题背景

社交媒体爆炸式增长使多模态虚假新闻（融合图像与文本）成为信息传播的核心威胁。现有基于端到端深度学习的检测方法（如多模态 Transformer）面临两大根本缺陷：

1. **动态语义演化难以捕捉**：静态模型无法适应突发社会事件衍生的新谣言变体
2. **分析异构性无法适应**：固定架构无法自适应虚假新闻的类型差异（如政治谣言 vs. 健康谣言 vs. 娱乐八卦）

### 1.2 核心动机

虚假新闻并非孤立内容，而是**社会认知的扭曲投射**，其检测需突破传统模型的封闭性，拥抱开放世界知识与动态推理。

**需要解决的两个核心挑战**：

| 挑战 | 描述 | 解决思路 |
|------|------|----------|
| **上下文稀疏性** | 虚假新闻的语义高度依赖隐式社会背景（如"某地爆炸"谣言需知晓真实地理事件），LLM 孤立分析时容易产生幻觉 | **以检索为认知锚点**：通过跨模态检索注入实时社会语境，将知识库转化为 LLM 的"外部记忆" |
| **分析异构性** | 不同类型虚假新闻需定制化分析维度（政治谣言依赖事实核查，健康谣言依赖科学证据），固定工具链无法泛化 | **以元认知为决策中枢**：设计分层规划机制，让 LLM 先宏观诊断认知需求，再动态路由轻量工具 |

### 1.3 核心创新

MCTL 是一个 **LLM 驱动的 Tool-Learning 框架**，完全摒弃传统训练范式，仅利用多模态表征（CLIP 联合嵌入）作为输入特征，通过 **检索增强生成（RAG）** 与 **元认知规划（Meta-Cognitive Planning）** 动态编排工具链。核心创新在于：

- 将 LLM 能力封装为可路由的工具集
- 构建 **DAG（有向无环图）** 驱动的推理流水线
- 实现 **认知-执行双层架构**：上层元认知规划器生成任务 DAG，下层工具路由器执行 DAG 节点
- 决策全链路透明可解释

## 二、整体架构

### 2.1 完整 Pipeline

```
INPUT (新闻文本 + 配图)
    │
    ▼
┌─────────────────────────────────────────────┐
│  Module 1: CAMR（上下文增强多模态检索）        │
│  CLIP编码 → FAISS检索 → Top-K相似样本上下文    │
└──────────────────────┬──────────────────────┘
                       │ 检索上下文 + 标签分布
                       ▼
┌─────────────────────────────────────────────┐
│  Module 2: MCP（元认知规划）                   │
│  LLM诊断认知需求 → 输出分析维度DAG             │
└──────────────────────┬──────────────────────┘
                       │ DAG执行计划
                       ▼
┌─────────────────────────────────────────────┐
│  Module 3: ATR（自适应工具路由）               │
│  8个专门LLM工具 → 按DAG拓扑序执行 → 短路机制   │
└──────────────────────┬──────────────────────┘
                       │ 各工具 (推理轨迹, 置信度, 判断)
                       ▼
┌─────────────────────────────────────────────┐
│  Module 4: MPRE（多视角推理集成）              │
│  共识识别 → 冲突仲裁 → 加权综合                │
│  [含共识直通机制：工具一致时跳过]               │
└──────────────────────┬──────────────────────┘
                       │ 综合判断 + 推理摘要
                       ▼
┌─────────────────────────────────────────────┐
│  Module 5: CBDF（基于共识的决策融合）          │
│  检索标签先验 + 不确定性评估 → 最终裁决         │
│  [含bypass机制 + 人工审核标记]                 │
└──────────────────────┬──────────────────────┘
                       │
                       ▼
OUTPUT (fake/real + 置信度 + 完整推理链)
```

## 三、五大模块详细设计

### 3.1 Module 1: CAMR（Context-Augmented Multimodal Retrieval）

**解决的问题**：虚假新闻的隐式语义无法从孤立输入推断，LLM 缺乏外部事实参照导致幻觉式判断。

**设计思路**：将图像-文本对转换为 CLIP 联合嵌入，在预构建知识库中检索最相似的 Top-K 样本作为上下文参考。

**技术细节**：

| 组件 | 实现 |
|------|------|
| 编码模型 | CLIP ViT-B/32（文本 + 图像联合编码） |
| 特征融合 | 文本-图像加权融合：`fused = 0.6 × text_emb + 0.4 × image_emb`，L2 归一化 |
| 检索引擎 | FAISS IndexFlatIP（余弦相似度） |
| 知识库 | 训练集全量样本（含文本、标签、事件上下文） |
| 默认 K | 5 |
| 缓存机制 | 基于 MD5 哈希（数据集 + text_weight）的磁盘缓存，避免重复编码 |

**输出**：结构化上下文包 `{retrieved_texts, labels, similarities, event_contexts}`

**核心价值**：为 LLM 提供「锚点参考」，显著减少幻觉式判断。例如，输入"某市地铁发生严重追尾事故！"，检索返回同日官方"运营正常"通告，即可揭示虚假属性。

### 3.2 Module 2: MCP（Meta-Cognitive Planning）

**解决的问题**：LLM 难以自主判断虚假新闻分析维度，导致工具冗余或缺失，千篇一律的分析策略无法适应不同类型新闻。

**设计思路**：模拟人类"**先想再看**"的认知模式——LLM 先基于检索上下文诊断该新闻需要哪些分析角度，生成任务 DAG 蓝图，再据此执行。

**两阶段工作流**：

**阶段 1 — 规划**：LLM 从 8 个预定义语义维度中选择 3-5 个最相关的，并定义执行顺序（DAG 边表示依赖关系）。

8 个语义维度：

| 维度 | 含义 | 映射工具 |
|------|------|----------|
| `factual_verification` | 事实核查 | knowledge_grounding |
| `emotional_tone` | 情绪操纵检测 | emotional_manipulator |
| `source_credibility` | 来源可信度 | evidence_comparator |
| `rhetorical_analysis` | 修辞手法分析 | rhetorical_scanner |
| `cross_modal_consistency` | 图文一致性 | cross_modal_aligner |
| `logical_coherence` | 逻辑连贯性 | inconsistency_amplifier |
| `temporal_context` | 时效性验证 | expectation_deviator |
| `comparative_evidence` | 对比证据分析 | evidence_comparator |

**规划输出格式**（JSON DAG）：
```json
{
  "semantic_layers": ["factual_verification", "rhetorical_analysis", "comparative_evidence"],
  "tool_dag": [
    {"tool": "factual_verification", "deps": []},
    {"tool": "rhetorical_analysis", "deps": []},
    {"tool": "comparative_evidence", "deps": ["factual_verification"]}
  ]
}
```

**阶段 2 — 执行**（Exp2 中由 LLM 按 DAG 逐维度分析；Exp3+ 由专门工具替代）

**核心价值**：Precision 大幅跃升（+22.2%），结构化思考有效避免了"仓促判断"。代价是 API 调用翻倍（1→2 次/样本）。

### 3.3 Module 3: ATR（Adaptive Tool Router）

**解决的问题**：通用 LLM 分析缺乏专门性，工具间存在隐式依赖（如"不一致性检测"需先完成"知识验证"），静态路由导致逻辑断裂。

**设计思路**：将 MCP 规划的抽象维度映射到 8 个**专门化 LLM 工具**，按 DAG 拓扑序动态调度执行。每个工具本质上是一个专门化的 LLM prompt，输入统一为 `(query_text, image, retrieved_context)`，输出标准化为 `(analysis_trace, confidence_score, judgment)`。

**8 个专门工具**：

| 工具 | 职责 |
|------|------|
| **Semantic Dissector** (语义解构器) | 分析文本字面/隐含义，识别双关语、隐喻、隐藏暗示 |
| **Rhetorical Scanner** (修辞探测器) | 识别反讽、夸张、标题党、煽动性语言、宣传手法 |
| **Knowledge Grounding** (知识锚定器) | 核查事实与检索事件一致性，验证人名/日期/事件/统计 |
| **Expectation Deviator** (期望偏移器) | 计算图文预期偏差，识别诱导点击和误导性框架 |
| **Cross-Modal Aligner** (跨模态对齐器) | 评估文本-图片语义一致性，判断图片是否脱离原始上下文 |
| **Inconsistency Amplifier** (不一致放大器) | 系统量化文本-图片-检索结果间的矛盾 |
| **Emotional Manipulator Detector** (情感操纵检测器) | 检测恐惧诱导、愤怒触发、紧迫感制造等情感操纵元素 |
| **Evidence Comparator** (证据对比器) | 对比目标新闻与检索样本的标签偏移和模式匹配 |

**关键机制**：

- **DAG 拓扑调度**：入度为 0 的工具先执行，前序工具输出注入后序工具 prompt（依赖注入）
- **层级并行执行**：同一 DAG 层级内的无依赖工具并行调用
- **短路机制**：若某工具 confidence ≥ 0.9 且已有 ≥2 个工具执行，跳过后续低优先级工具
- **标准化输出解析**：从 LLM 响应中提取 `(Analysis, Confidence, Judgment)` 三元组

---

### 3.4 Module 4: MPRE（Multi-Perspective Reasoning Ensemble）

**解决的问题**：单一工具视角片面（如仅文本分析忽略视觉误导），需融合异构证据并仲裁冲突。

**设计思路**：收集 ATR 执行的所有工具输出，由 LLM 进行**证据加权聚合**（而非简单投票）——识别共识、仲裁冲突、生成加权摘要。

**MPRE 三步工作流**：

1. **共识识别**：找出多工具一致的发现（如多个工具同时标记图文冲突）
2. **冲突仲裁**：对分歧点判断哪个工具更可靠（基于置信度、推理质量、维度相关性）
3. **加权综合**：高置信度发现权重更大，共识证据优先于孤立发现

**关键调优（来自 Exp4 调优记录）**：

MPRE 初始版本导致 Macro-F1 **下降 7.44%**（0.8636→0.7892），经诊断发现三个问题并逐步解决：

| 问题 | 诊断 | 解决措施 |
|------|------|----------|
| 低置信度噪声 | 置信度 <0.5 的工具（3.7%）污染 MPRE 输入 | 设置 `CONFIDENCE_FILTER_THRESHOLD = 0.5`，过滤低置信度证据 |
| MPRE 推翻多数票频繁出错 | 推翻多数票时错误率 60%（vs 同意时 11.8%） | 注入**置信度加权投票信号**作为统计锚点 |
| 73% 样本无需仲裁 | 工具一致性高的样本调用 MPRE 反而引入不确定性 | **共识直通机制**：所有高置信度工具一致 + margin ≥ 0.5 时跳过 MPRE |

**核心结论**：MPRE 的价值不在于对每个样本融合，而在于**精准仲裁真正的冲突**。短路阈值调整为 0.95（比 Exp3 的 0.9 更宽松），确保至少 2 个工具参与。

---

### 3.5 Module 5: CBDF（Consensus-Based Decision Fusion）

**解决的问题**：多视角推理可能矛盾（语义工具判"真实"但情感工具判"虚假"），需引入统计先验进行鲁棒决策。

**设计思路**：在 MPRE 证据摘要基础上，引入检索样本的**标签分布**作为统计先验信号，结合**不确定性评估**，做出最终裁决。

**CBDF 决策规则**：

| 条件 | 动作 |
|------|------|
| MPRE 判虚假 + 检索多数虚假 | 强虚假信号，高置信度判定 fake |
| MPRE 判真实 + 检索多数真实 | 强真实信号，高置信度判定 real |
| MPRE 与检索标签一致 | 遵循共识方向 |
| MPRE 与检索标签冲突 | 优先 MPRE（证据驱动），但标注冲突、降低置信度 |
| 高不确定性 + 低投票优势 | 标记为"需人工审核" |

**不确定性计算**（`compute_uncertainty()`）：
- 工具置信度标准差（`std`）
- 最大分歧度（`max_divergence`）
- 预测一致性（`pred_agreement`）
- 不确定性标志：`std > 0.15`（对应 ~30% 分歧范围）

**CBDF bypass 条件**：共识直通已触发 + 加权投票 margin ≥ 0.3 + 检索先验与共识方向一致 → 跳过 CBDF 调用。

**调优记录**：初始版本同样因对高确定性样本的过度干预导致 Macro-F1 下降 5.51%（0.8876→0.8325），应用 bypass 后恢复至 0.8884。

## 四、关键设计原则

### 4.1 共识直通（Consensus Bypass）— 通用架构原则

Exp4 和 Exp5 都验证了同一规律：**下游融合/决策模块只应在上游模块真正存在分歧时才介入**。当工具已高置信度达成共识时，额外的 LLM 调用不仅浪费资源，还会引入"过度思考"噪声。

实际效果：71.2% 样本走共识直通跳过 MPRE，71.2% 样本走 bypass 跳过 CBDF。

### 4.2 噪声隔离

低置信度证据在传递给下游前被过滤（阈值 0.5），确保 MPRE/CBDF 输入质量。

### 4.3 锚点引导

通过置信度加权投票信号为下游模块提供统计先验，避免无根据的反转。

### 4.4 依赖注入

前序工具输出作为后序工具的输入上下文，确保分析的逻辑连贯性。

### 4.5 按需介入

各模块均遵循"**该介入时介入，该放手时放手**"原则——MPRE 只处理冲突仲裁，CBDF 只在存在真正冲突时调用。

## 六、GossipCop 效果分析与改进

### 6.1 问题诊断

GossipCop 上 Accuracy 仅 77.46%（vs PolitiFact 91.26%），Fake-F1 仅 0.5238。核心问题是 **FP 过多**：大量真实新闻被误判为虚假。

**五大根因**：

| 根因 | 描述 |
|------|------|
| **数据不平衡 + 领域特殊性** | 80.5% 真实/19.5% 虚假；娱乐八卦新闻天然使用煽动性语言，与 prompt 中"虚假特征"重叠 |
| **工具级偏差** | rhetorical_scanner（60.33% acc, 58.68% fake 预测率）和 emotional_manipulator（58.06% acc, 79.84% fake 预测率）在娱乐新闻上几乎全部判 fake |
| **Bypass 错误传播** | 71.35% 样本走 bypass，但 136 个 bypass 错误占全部 284 错误的 47.9% |
| **检索标签噪声** | 12.4% 真实样本的 Top-5 检索多数为 fake，FP 率高达 51.59% |
| **模型能力受限** | gemini-2.5-flash-lite 对细粒度的娱乐领域真假区分能力不足 |

### 6.2 通用改进方案（Exp5 v2）

为避免数据集特定硬编码，设计了五项**通用**改进（无任何 `if dataset == "xxx"` 判断）：

| 改进 | 思路 | 通用性来源 |
|------|------|------------|
| **领域感知 MCP 规划** | MCP 增加 Step 0 领域自动识别，据此调整分析维度选择 | 基于新闻内容自动判断，不硬编码数据集规则 |
| **检索校准工具分析** | 工具先观察检索到的真实样本作为领域"正常"基线，只标记偏离基线的特征为可疑 | 基线从检索数据动态获取 |
| **动态工具可靠性** | 运行时基于检索先验对齐(±0.3)、工具间一致性(±0.1)、过度自信惩罚(-0.2)自动算权重 | 不依赖预设权重表 |
| **保守 Bypass 策略** | 最少工具数 2→3，margin 阈值 0.5→0.6，必须检索先验对齐 | 通用容错性改进 |
| **基率感知 MPRE/CBDF** | 注入检索基率感知指令，引导贝叶斯推理 | 检索基率由数据驱动自动适应 |

### 6.3 领域定制改进方案（分阶段）

| 阶段 | 改动 | 预期提升 |
|------|------|----------|
| **Phase 1** (Prompt层) | GossipCop 专用领域 prompt + 工具 prompt 领域校准 + 禁用 emotional_manipulator | +5-10% Accuracy |
| **Phase 2** (机制层) | 工具可靠性加权投票 + 收紧 bypass 条件 + MPRE/CBDF 领域校准 | +3-5% |
| **Phase 3** (模型层) | 升级模型 + 增大 K + Few-shot 示例注入 | 需实验验证 |

## 十、核心结论

1. **MCTL 通过五个渐进式模块，将虚假新闻检测 Macro-F1 从 0.7453 提升至 0.8884（+19.2%）**

2. **CAMR + MCP 是框架基石**：检索增强和结构化思考贡献了 74% 的总提升，是最关键的能力

3. **ATR 实现专门化分工和自适应执行**：8 个 LLM 工具按 DAG 拓扑序动态调度，短路机制控制开销

4. **MPRE + CBDF 实现精准冲突仲裁**：核心价值不在于处理每个样本，而在于精准识别并仲裁真正的分歧

5. **共识直通是通用架构原则**：在 MPRE 和 CBDF 上均验证——下游模块只在上游存在真正分歧时才介入

6. **领域适应是下一步重点**：GossipCop 的效果差距表明，不同新闻领域需要不同的分析校准策略，通用改进方案（领域感知 MCP、检索校准、动态工具权重）是解决方向

7. **完全无需训练**：框架仅依赖 LLM 的零样本推理与 CLIP 多模态表征，部署轻量且可灵活切换底座模型
